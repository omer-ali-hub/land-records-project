What this is

This is a static dashboard hosted via GitHub Pages. The browser loads summary.json from the same folder and renders tables and totals.

How to update the data

Run python scripts/build_summary.py locally. That script writes data_summary/summary.json (for analysis) and copies the same file to docs/summary.json (for the website).

Parcel data page

The same script scans your local parcel_data folder (one subfolder per county)—by default ~/Library/CloudStorage/Dropbox/parcel_data or ~/Dropbox/parcel_data depending on which exists—and writes parcel_counties.json. The Dropbox share link in the UI is for people only. Override PARCEL_DATA_DIR in scripts/build_summary.py if your path differs.

Counties page · data folder link

Row counts come from files under DATA_ROOT in scripts/build_summary.py. The button uses a Dropbox shared-folder HTTPS link (see DROPBOX_DATA_FOLDER_URL), like the parcel page. If that constant is empty, the build falls back to a local file:// path.

How to turn on the public link (first time only)

GitHub does not publish the site until you tell it which folder to use. “ main + /docs” means: use the docs folder on the main branch as the website root (so docs/index.html becomes the home page).

  1. Open your repo on GitHub → Settings (top bar of the repo, not your profile).
  2. Left sidebar → Pages.
  3. Under Build and deployment → Source, choose Deploy from a branch.
  4. Under Branch: select main, folder /docs, then Save.
  5. Wait one to two minutes, then open https://omer-ali-hub.github.io/land-records-dashboard/. If it still 404s, refresh after another minute.

How to publish updates

Commit and push changes under docs/ (especially docs/summary.json and docs/parcel_counties.json) to main. GitHub Pages updates shortly after each push.
